 Sets the state of the audio link (radio link) between a base/dongle and a wireless headset. When an audio link is open, all sounds from the PC will be audible in the headset and audio from your headset microphone will be available to PC applications. An audio link is opened automatically whenever you make or receive a call and closed again when you hang up. Use this command if you would like to listen to music or other audio when you are not on a call. Changes in the audio link state is reported as StateChangeEvent.AudioLinkOpen events. More...
 
void SetCommunicationLinkState (bool linkOpen)

When a communication link is open, all sounds from the PC will be audible in the headset and audio from your headset microphone will be available to PC applications. A communication link is typically used when you want to join a web conference. It can also be used for manual control of the headset's hook state when it is used with an unsupported softphone. Changes in the communication link state is reported as StateChangeEvent.CommunicationLinkOpen events. More...

Lock device. A lock must have been obtained prior to calling any of the Set... methods (SetHookState, SetMicrophoneMute, SetRinger, SetOnHold).

A softphone client must lock a device before it initiates a call. This is to ensure that other softphone clients do not interfere while the call is in progress. When the call is terminated the client must call Unlock() to the unlock the device so it becomes available to other softphone clients. In case the lock could not be obtained, the softphone client should not put the call through.

When using out-band ringtone you may hear both a softphone ringtone and and a local ringtone in the headset.

◆ DeviceRemoved

EventHandler JabraSDK.IDevice.DeviceRemoved

The DeviceRemoved event is raised when the physical device represented by this object has been removed (unplugged) from the PC. After receiving this event a DeviceRemovedException will be thrown on device access.

When the physical device has been removed it is the object creator's responsibility to call Dispose() on the object. Note: Do not call Dispose synchronously from a DeviceRemoved event handler.

Note: This event handler is deprecated.
