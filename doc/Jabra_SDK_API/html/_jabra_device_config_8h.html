Restore factory settings to device. Note that if used on a device connected via dongle (or directly via BT), the pairing list in the device will be cleared, and the connection to the device will be lost.

◆ Jabra_GetSetting()

LIBRARY_API DeviceSettings * Jabra_GetSetting ( unsigned short  deviceID,
const char *  guid 
)

The proposed design is to have the memory allocated at the SDK level as the user may not aware of how much memory needs to be allocated. Settings vary up on connected devices. So it is good SDK to decide how much memory to allocate for settings.

Gets the unique setting identified by a GUID of a device.

Parameters
[in]deviceIDDevice ID of the device to read/get the setting from.
[in]guidThe unique setting identifier, the string must be zero terminated.
Returns
Pointer to the structure containing the requested setting. In case the device is not found or could not be accessed a NULL pointer is returned.
Note
As memory is allocated through the SDK it needs to be freed by calling Jabra_FreeDeviceSettings.

◆ Jabra_SetSettingsChangeListener()

LIBRARY_API Jabra_ReturnCode Jabra_SetSettingsChangeListener ( unsigned short  deviceID,
SettingsListener  listener,
const DeviceSettings *  settings 
)

Add a listener for changes to the device settings. The listener will be called when a setting is changed in the device, regardless of who changed it (this instance, another instance, or the user through MMI).

Parameters
deviceIDID for a specific device.
listenerThe function to receive callbacks. Set to null to cancel listening.
settingsThe settings which the client is interested in. Remains owned by the caller. Set to null to cancel listening.
Returns
Return_Ok if successful.
Not_Supported if feature is not supported.
Device_Unknown if device not known.
