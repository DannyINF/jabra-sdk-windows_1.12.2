Detailed Description

Defines the interface for querying constants defined for a specific device.

The constants can be of different types: integers, booleans, strings, as well as composed types: structs and lists.

The purpose of the constants are to supply values that are handy for applications, but not directly required–and hence not available–by the devices. The constants are accessed by keys, which are simply strings, and these are obtained from the documentation.

Note
A key will never be reused for a constant of a different type, and the key will mean the same for all devices where it is defined.
